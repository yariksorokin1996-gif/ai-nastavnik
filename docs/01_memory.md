# 1. ПАМЯТЬ — Система сбора, хранения и обновления информации о пользователе

## Принцип
Бот должен знать пользователя как лучший друг — помнить людей, события, паттерны, предпочтения.
Не «база данных», а «друг, который знает тебя».

---

## 1.1. Архитектура: 3 слоя памяти

### Слой А — Семантический профиль (всегда в контексте, ~300-500 слов)

Компактная выжимка о человеке, подставляется в КАЖДЫЙ промпт:

```
=== ПРОФИЛЬ ===
Имя, возраст, город
Семья: роль — имя (подробности)
Работа / занятость
Главная проблема (зачем пришла)
Корневой паттерн (глубинная причина)
Текущая цель + дата постановки
Стиль общения (что работает / что раздражает)
Триггеры (что → какая эмоция)
Сильные стороны (что сама не замечает)
Ключевые достижения (с датами)
⚠️ Чувствительные темы (не поднимать первым)
```

### Слой Б — Эпизодическая память (подтягивается по релевантности)

Конспекты разговоров + ключевые события. В промпт попадают только 3-5 релевантных эпизодов.

**Формат конспекта:**
```
Дата, время, длительность
Тема разговора
Эмоциональный тон (начало → конец)
Ключевой инсайт сессии
Обязательства / эксперименты
Что сработало / что не сработало
```

**Формат ключевого события:**
```
Дата + событие + контекст + эмоция + связь с паттерном
Пример: "22.01 — впервые не ответила клиенту сразу.
         Тряслась 3 часа, но выдержала.
         Связано с целью: границы. Прогресс."
```

### Слой В — Процедурная память (как работать с этим человеком)

```
✅ Что работает: конкретные техники, типы вопросов
❌ Что не работает: что раздражает, что игнорирует
📏 Предпочтения: длина ответов, стиль, время активности
💡 Лучший момент для инсайта: при каких условиях раскрывается
```

---

## 1.2. Что именно бот собирает

### Из каждого сообщения (пассивно, Claude Haiku):

| Категория | Примеры | Как извлекается |
|---|---|---|
| Факты о жизни | Работа, семья, город, возраст | Из контекста речи |
| Люди | Имена, роли, качество отношений | Упоминания в разговоре |
| Эмоции | Текущее состояние | Тон сообщения |
| Паттерны | Повторяющиеся слова/мысли | Regex-детектор + LLM каждые 5 сообщений |
| Обязательства | Что обещала сделать | LLM-извлечение |
| Инсайты | Моменты осознания | LLM определяет «ага-моменты» |

### После каждого разговора (автоматически, Claude Haiku):

На вход: полный разговор + текущий профиль → на выход JSON:
```json
{
  "new_facts": [
    {"subject": "user", "fact": "...", "confidence": "high/medium/low", "sensitive": false}
  ],
  "updated_facts": [
    {"old": "...", "new": "...", "reason": "..."}
  ],
  "people_updates": [
    {"name": "...", "role": "...", "change": "..."}
  ],
  "session_summary": "...",
  "emotional_arc": "тревога → облегчение",
  "techniques_worked": ["..."],
  "techniques_failed": ["..."],
  "new_patterns": [],
  "connections": [
    {"from": "X", "to": "Y", "type": "причина/прогресс/триггер"}
  ]
}
```

## 1.3. Карта связей (люди + темы + паттерны)

Бот строит граф связей между сущностями:

```
Мама → критикует профессию → триггерит "я не достойна"
       конфликты обостряются перед праздниками

Муж (Саша) → не понимает → чувство одиночества
              конфликты из-за денег → связано с нестабильным доходом

Работа (фриланс) → клиент Олег → не может сказать "нет" → страх отвержения
                    нестабильный доход → тревога → ссоры с мужем

Корневой паттерн: страх отвержения
  ├── проявляется с клиентами (не может отказать)
  ├── проявляется с мамой (не может возразить)
  └── корень: мама всю жизнь говорила "ты не справишься"
```

Связи позволяют боту делать глубокие инсайты: «Это не про Олега. Это тот же страх, что и с мамой».

---

## 1.4. Уровни уверенности

Каждый факт сохраняется с меткой:

| Уровень | Когда | Как бот использует |
|---|---|---|
| **Точно** | Прямо сказала: «мне 32, работаю дизайнером» | Ссылается свободно |
| **Скорее всего** | Из контекста: «забрала из сада» → есть ребёнок | Ссылается через вопрос: «ты упоминала сад — у тебя есть дети?» |
| **Неясно** | Двусмысленно: «бывший друг» | НЕ ссылается, ждёт уточнения |

**Бот НИКОГДА не додумывает.** Не уверен — спрашивает или молчит.

---

## 1.5. Разделение: факты vs эмоции vs паттерны

| Тип | Как хранить | Пример |
|---|---|---|
| **Факт** | Постоянно, обновляется при противоречии | «Работает дизайнером» |
| **Эмоция** | С датой и временем, затухает | «2 ночи, 15 янв: бессмысленность» |
| **Паттерн** | Только после 3+ повторений | «Обесценивает себя в стрессе» |

Бот НЕ говорит: «Ты считаешь себя ничтожеством»
Бот МОЖЕТ сказать: «Когда тебе тяжело — ты начинаешь обесценивать себя. Это паттерн, не правда о тебе»

---

## 1.6. Правила обращений к людям

**Принцип: бот называет человека ТАК, КАК ОНА САМА его называет**

| Роль | В разговоре | Имя |
|---|---|---|
| Мама/папа | Всегда «мама»/«папа» | Хранится, НЕ используется |
| Бабушка/дедушка | «бабушка»/«дедушка» | Хранится, НЕ используется |
| Муж/парень | По имени | «Как Саша?» |
| Ребёнок | По имени | «Как Алиска?» |
| Сестра/брат (один) | По роли | «Как сестра?» |
| Сёстры/братья (несколько) | По именам | Чтобы различать |
| Подруги/друзья | По имени | «Как Настюха?» |
| Коллеги/клиенты | По имени | «Как с Олегом?» |
| Начальник | Как она сама называет | «шеф»/по имени |

Бот запоминает ЕЁ форму обращения: если говорит «Сашка» — бот тоже «Сашка», не «Александр».

---

## 1.7. Чувствительная информация

Темы, автоматически помечаемые как чувствительные:
- Насилие, аборт, зависимости, суицидальные мысли, измены, серьёзные болезни, финансовые долги

**Правило:** бот НИКОГДА не поднимает чувствительную тему первым. Ссылается только если она сама вернулась к теме.

---

## 1.8. Механизм обновления

### А) Противоречие (мгновенно):
- Новый факт противоречит старому → новый сохраняется с пометкой «требует подтверждения»
- Старый помечается как «возможно устарел»
- Бот при случае уточняет в контексте разговора

### Б) Естественная проверка (периодически):

| Тип факта | Частота проверки | Как |
|---|---|---|
| Работа | Каждые 4 недели | Упоминает в контексте |
| Отношения | Каждые 4 недели | «Как у вас с Сашей?» |
| Цель | Каждые 2 недели | «Как продвигается [цель]?» |
| Активный конфликт | Каждую неделю | «Разрешилась ситуация с...?» |
| Настроение | Не проверять | Устаревает само |

### В) Затухание (автоматически):

| Тип информации | Срок актуальности |
|---|---|
| Текущее настроение | 1 день |
| Активный конфликт | 2 недели |
| Цель | 3 месяца |
| Работа/отношения | 6 месяцев |
| Паттерн | Не устаревает (но может быть «проработан») |
| Имена людей | Не устаревает |

---

## 1.9. Явные запросы пользователя

| Запрос | Реакция бота |
|---|---|
| «Запомни, что...» | Сохранить с максимальной уверенностью. Подтвердить кратко: «Запомнила» |
| «Это не так» / коррекция | Мгновенное обновление + короткое: «Прости, исправила» |
| «Забудь это» / «Не напоминай» | Пометить как скрытое (не удалять). «Хорошо, не буду» |
| «Ты помнишь, что...?» | Если помнит — процитировать конкретику. Если нет — честно: «Не помню деталей, расскажешь?» |

**Никогда не выдумывать.** Лучше честно не помнить.

---

## 1.10. Подстановка в контекст (что попадает в промпт)

Перед каждым ответом формируется контекст:

1. **Всегда:** семантический профиль (слой А, ~300-500 слов)
2. **Всегда:** процедурная память (слой В, ~100-150 слов)
3. **По релевантности:** 3-5 эпизодов из слоя Б

Выбор релевантных эпизодов: Claude Haiku читает последние 2 сообщения + заголовки всех конспектов → выбирает самые релевантные.

---

## 1.11. Три типа обратных ссылок (как бот использует память)

**1. Прогресс** (самый мощный):
> «В январе ты не могла сказать "нет". Сегодня ты уволила клиента. Видишь разницу?»

**2. Паттерн через время:**
> «Ты сейчас говоришь про нового клиента то же, что говорила про Олега — тот же страх»

**3. Детали жизни:**
> «Как Алиска? Привыкла к саду?»

---

## 1.12. Масштабирование (год использования)

### Что накапливается за год (~180 разговоров):
- Профиль: ~500 слов (уплотнённый)
- Конспекты: 180 штук
- Месячные резюме: 12 штук
- Ключевые события: ~50
- Карта людей: ~20-30 человек
- Все сообщения: ~2500 (архив)
- Паттерны: ~15-20

### Хранение: база данных на сервере (SQLite → PostgreSQL)

### Уплотнение (раз в месяц):
1. Месячное резюме: главные темы, прогресс, откаты, инсайты (~200 слов)
2. Обновление профиля: устаревшие факты → история, новые обобщения → профиль
3. Архивация: эпизоды старше 3 месяцев, вошедшие в обобщение → архив

### Что в «голове» бота при каждом ответе:
```
├── Профиль: ~500 слов
├── Как с ней работать: ~150 слов
├── 3-5 релевантных конспектов: ~500 слов
├── Текущий разговор: последние 15-20 сообщений
└── Итого: ~2000-3000 слов контекста памяти
```
